# Name Counter

![Go Version](https://img.shields.io/badge/Go-1.26+-00ADD8?style=flat&logo=go)
![Build Status](https://img.shields.io/badge/build-passing-brightgreen)

Модульная CLI-утилита, написанная на Go, для подсчета частоты встречаемости имен в больших текстовых файлах и их удобной
сортировки.

## Обзор

Приложение `name-counter` построчно читает текстовый файл (одно имя на строке) и подсчитывает, сколько раз встречается
каждое уникальное имя. Утилита спроектирована для эффективной работы с большими объемами данных: она использует пакетное
чтение (batch processing) и агрегацию в оперативной памяти (in-memory), а также предоставляет гибкие настройки
сортировки результата.

## Ключевые возможности

* **Высокая производительность:** Использование `bufio.Scanner` для потокового ввода-вывода и обработка данных батчами (
  порциями) минимизируют накладные расходы на аллокацию памяти.
* **Модульная архитектура:** Проект построен на принципах чистой архитектуры, отделяя бизнес-логику (подсчет,
  сортировка) от интерфейса командной строки (CLI).
* **Гибкая сортировка:** Поддержка нескольких режимов вывода (`desc`, `asc`, `none`) с использованием современных
  дженериков `slices.SortFunc` (Go 1.21+).
* **Структурированное логирование:** Интеграция со стандартным пакетом `log/slog` для удобного и понятного вывода логов
  и ошибок.
* **Надежность:** Код покрыт как модульными (Unit) тестами для ядра, так и сквозными (E2E / Integration) тестами для
  собранного бинарного файла.

## Архитектурные решения

1. **In-Memory Агрегация:**
   Учитывая, что в мире существует около 1.5 млн уникальных имен (в среднем 7-10 символов), использование
   `map[string]int` в худшем случае потребует не более 1-2 ГБ оперативной памяти. Для локальных данных (например, города
   с населением 5.5 млн человек, но с ограниченным набором имен) расход памяти составит менее 500 МБ. Таким образом,
   агрегация в RAM является наиболее быстрым и оправданным решением.
2. **Батчинг (Batch Processing):**
   Для оптимизации работы процессора (локальность кэша) данные считываются настраиваемыми порциями (`--batch-size`).
   Приложение накапливает срез строк перед их отправкой в хэш-таблицу.
3. **Контрактный подход (Интерфейсы):**
   Базовые операции скрыты за интерфейсами `core.Counter` и `core.Sorter`. Это позволяет легко расширять функционал (
   например, добавить внешнюю сортировку на диске при необходимости) без изменения CLI-слоя.

## Установка и сборка

Склонируйте репозиторий и используйте `Makefile` для сборки:

```bash
git clone github.com/ilindan-dev/name-counter.git
cd name-counter
make build
```

Скомпилированный исполняемый файл будет сохранен в директории `./bin/`.

## Использование

Утилите требуется передать один позиционный аргумент - путь к текстовому файлу.

```Bash
./bin/name-counter [путь_к_файлу] [флаги]
```

Флаги:

- `-m`, `--mode string`: Режим сортировки результатов. Возможные варианты:
    - `desc` (по умолчанию): По убыванию частоты.
    - `asc`: По возрастанию частоты.
    - `none`: Без сортировки (вывод как есть из хэш-таблицы, максимальная скорость).

- `-b`, `--batch-size` `int`: Количество строк, считываемых в память за одну итерацию перед обновлением счетчиков. (по
  умолчанию `1000`)
  `-h`, `--help`: Показать справочную информацию.

## Примеры

### Стандартный запуск (сортировка по убыванию):

```Bash
./bin/name-counter data/names.txt
```

### Сортировка по возрастанию с кастомным размером батча:

```Bash
./bin/name-counter data/names.txt --mode asc --batch-size 5000
```

### Вывод без сортировки:

```Bash
./bin/name-counter data/names.txt --mode none
```
